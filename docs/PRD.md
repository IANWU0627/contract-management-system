# Toy Contract - 企业合同管理系统产品需求文档

## 1. 项目概述

### 1.1 产品定位
Toy Contract 是一款面向企业的合同全生命周期管理平台，提供从合同创建、审批、签署到归档的一站式解决方案。系统旨在通过数字化手段提升企业合同管理效率，降低合同风险，规范审批流程，为企业决策提供数据支持。

### 1.2 目标用户
- **企业法务部门**：负责合同起草、审核、管理，需要专业的合同管理工具
- **采购/销售部门**：频繁与外部签订合同，需要快速创建和跟踪合同
- **行政管理人员**：负责合同归档和管理，需要高效的文档管理功能
- **企业高管**：需要合同数据概览和决策支持，需要直观的统计报表

### 1.3 核心价值
- **提升合同管理效率**：数字化管理，减少纸质文档，自动化流程处理
- **降低合同风险**：规范化审批流程，关键节点把控，风险预警机制
- **规范审批流程**：自定义审批节点，流程透明化，责任明确化
- **数据可视化分析**：多维度统计报表，趋势分析，决策支持
- **促进团队协作**：合同评论、版本历史，提高团队沟通效率

## 4. 安全特性

### 4.1 认证与授权
- [x] **JWT Token认证**：无状态认证，支持Token刷新，防止CSRF攻击
- [x] **Token自动刷新**：Access Token 2小时，Refresh Token 7天，无感续期
- [x] **BCrypt密码加密**：强度10，安全存储，防止密码泄露
- [x] **旧密码自动升级**：登录时自动升级旧密码哈希算法
- [x] **基于角色的访问控制 (RBAC)**：细粒度权限管理，支持动态权限分配
- [x] **登录安全**：登录失败次数限制，IP锁定，登录验证码（待实现）

### 4.2 API安全
- [x] **速率限制**：60次/分钟，防止暴力攻击，保护API服务
- [x] **CORS安全配置**：白名单机制，防止跨域攻击
- [x] **文件路径遍历防护**：防止路径遍历攻击，安全文件访问
- [x] **文件类型白名单验证**：仅允许上传安全的文件类型
- [x] **输入验证**：参数校验，SQL注入防护，XSS防护
- [x] **敏感信息保护**：敏感信息脱敏，HTTPS传输

### 4.3 审计日志
- [x] **AOP自动记录操作**：切面编程，自动记录所有操作
- [x] **详细记录内容**：用户、时间、IP地址、操作类型、模块、操作结果、操作详情
- [x] **日志存储**：数据库存储，支持查询和导出，日志不可篡改
- [x] **日志分析**：操作趋势分析，异常操作检测，安全审计

## 5. API 接口规范

### 5.1 响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1711843200000
}
```

### 5.2 错误码定义
| 错误码 | 说明 | 处理建议 |
|--------|------|----------|
| 200 | 成功 | 正常处理响应数据 |
| 400 | 请求参数错误 | 检查请求参数格式和值 |
| 401 | 未认证 | 重新登录获取Token |
| 403 | 无权限 | 检查用户权限 |
| 404 | 资源不存在 | 检查资源ID是否正确 |
| 500 | 服务器内部错误 | 联系系统管理员 |
| 429 | 请求过于频繁 | 减少请求频率 |

### 5.3 分页参数
```
GET /api/contracts?page=1&pageSize=10&sort=createTime&order=desc&type=PURCHASE&status=APPROVED
```

### 14.2 部署计划

#### 14.2.1 环境准备
| 环境 | 配置要求 | 准备事项 |

#### 14.2.2 部署架构
```
┌─────────────────────────────────────────────────────────────┐
│                       生产环境                               │
┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐
│  Nginx负载  │  │  Nginx负载  │  │       数据库集群       │
│  均衡器     │  │  均衡器     │  │  (MySQL主从复制)      │
└─────┬──────┘  └─────┬──────┘  └─────────────────────┘
      │               │                       │
┌─────▼──────┐  ┌─────▼──────┐  ┌───────────▼───────────┐
│  应用服务器  │  │  应用服务器  │  │  主库          从库  │
│  (节点1)    │  │  (节点2)    │  │                   │
└────────────┘  └────────────┘  └─────────────────────┘
```

#### 14.2.3 部署步骤
1. **服务器准备**
   - 购买云服务器或物理服务器
   - 安装操作系统（CentOS 7+/Ubuntu 20.04+）
   - 配置网络和防火墙

2. **环境配置**
   - 安装JDK 21
   - 安装Node.js 18+
   - 安装Maven 3.8+
   - 安装MySQL 8.x
   - 安装Nginx

3. **应用部署**
   - 构建前端：`npm run build`
   - 构建后端：`mvn clean package -DskipTests`
   - 部署前端静态资源到Nginx
   - 部署后端应用到应用服务器
   - 配置Nginx反向代理

4. **数据库部署**
   - 创建数据库：`CREATE DATABASE contract_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   - 导入初始数据：执行data.sql
   - 配置主从复制

5. **系统配置**
   - 配置环境变量
   - 配置JWT密钥
   - 配置邮件服务器
   - 配置CORS白名单

### 14.4 运维策略

#### 14.4.1 日常运维

- **日志**：使用ELK Stack收集和分析日志
- **备份**：每日全量备份，每小时增量备份
- **更新**：定期更新系统补丁和依赖

#### 14.4.2 故障处理
| 故障类型 | 可能原因 | 处理方法 |
|----------|----------|----------|
| **服务不可用** | 服务器宕机，网络故障 | 检查服务器状态，重启服务 |
| **数据库连接失败** | 数据库宕机，连接池耗尽 | 检查数据库状态，重启数据库 |
| **API响应缓慢** | 服务器负载过高，数据库查询慢 | 检查服务器负载，优化数据库查询 |
| **文件上传失败** | 磁盘空间不足，权限问题 | 检查磁盘空间，调整权限 |
| **邮件发送失败** | 邮件服务器配置错误 | 检查邮件服务器配置 |

#### 14.4.3 性能优化
- **数据库优化**：定期分析慢查询，优化索引
- **服务器优化**：调整JVM参数，优化内存使用
- **缓存策略**：使用Redis缓存热点数据
- **负载均衡**：增加服务器节点，分散负载
